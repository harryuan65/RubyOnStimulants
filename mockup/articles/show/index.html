Postgres: Query string

✎
tags: postgresql , db harryuan.65 Mon, 26 Oct 2020

Scenario

前人建了一個Courses Table，其中有一個欄位叫做supported_languages，他是一個string (varchar(255))

courses

id name supported_languages
143 Ruby on Rails: 進階課程 2

145 Postgresql: Хорошая база данных 1,3,11

users

Document

假設公司文件規定數字、語言對應：

[
  {1: "English"},

需求？

GET /courses/index 要回傳符合用戶語言的課程。

假設current_user是小明，他需求語言是 "2"，所以我們要回傳143,145,146的課程給他。

我心想：是要怎樣query string 中的數字？

First attempt

第一個想到就是直接用pg的pattern matching。

列了一下各種可能，這些都是我要的：
"2", "1,2", "2,3", "1,2,3" 

select * from courses
where support_languages ~ '(^2$|,2$|^2,|,2,)'

results:

id name supported_languages
143 Ruby on Rails: 進階課程 2
144 Tensorflow for beginners 1,2,3,11
146 React: Powerful Library for Single Page Applications 1,2,3,8

看起來還行，但是如果我今天是김정은怎麼辦？

select * from courses
where support_languages ~ '(^1$|,1$|^1,|,1,)' or support_languages ~ '(^4$|,4$|^4,|,4,)'

如果要搜的語言越來越多，是不是query會越來越肥？這樣我就要動態去append一堆重複的條件上去。

詢問了一下我們data的專業大大傑克，他說可以用pg的json來亂兜。
蛤？亂兜？

原理

首先我們把 '1,2,3' 頭尾加上 '[', ']'，再轉成jsonb

select '[1,2,3]'::jsonb

接著我們使用postgres提供的contain operator @>
問他說'[1,2,3]'::jsonb 有沒有包含 '[2]'::jsonb

select (case when ('[1,2,3]'::jsonb @> '[2]'::jsonb) then true else false end)

欸靠腰真的回傳true欸

case
true

結束！YAY

select id, name, support_languages
from users
where ('[' || support_languages || ']')::jsonb @> '[1,8]'::jsonb

results:

id name supported_languages
144 Tensorflow for beginners 1,2,3,11
145 Postgresql: Хорошая база данных 1,3,11
146 React: Powerful Library for Single Page Applications 1,2,3,8
